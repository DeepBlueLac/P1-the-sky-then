# 此刻星空体验审计（2026-07-14）

## 审计范围

Web 首屏工作台：输入时刻与地点、查看实时海报、选择地点、导出 PNG。审计模式为 UX 与可访问性合并审计，证据来自本次 Playwright 运行的截图与交互。

## 用户目标与无障碍目标

用户应在不猜测下一步的情况下输入纪念时刻、选择地点、确认实时星图并下载海报。界面在 390px 移动端和 1440px 桌面端都应有可读文本、明确操作和可见焦点。

## 步骤与健康度

1. [桌面编辑工作台](01-desktop-editor.png)：健康。52/48 编辑器/预览结构清晰；星空视频只作背景，玻璃面板保持输入和海报可读。
2. [移动端预览优先](02-mobile-preview-first.png)：健康。海报与下载入口在首个内容面板出现，编辑器紧随其后；无横向溢出。
3. [Paris 地点选择](03-paris-selected.png)：健康。选择后输入值、坐标/时区、星点数量和海报地点同步更新。
4. PNG 导出：健康。Playwright 验证下载事件和文件名；导出实现跳过跨域展示字体抓取，避免展示字体影响海报 SVG 导出。

## 已确认的优点

- 视频背景、深色遮罩与半透明面板形成明确层级，不影响主任务。
- 按钮与输入框具备 hover、active、focus-visible 状态；`prefers-reduced-motion` 下会停止视频与过渡。
- 关键输入保持标签、错误提示、Enter 搜索和禁用状态；地点、重置、付费意向流程均有可恢复反馈。

## 风险与限制

- 截图不能证明所有读屏器朗读顺序或真实设备上的视频解码表现；现有 `accessibilityLabel`、语义角色与自动化交互为补充证据。
- 背景视频为远程资源，离线或被网络策略拦截时会自然退化为深色背景；核心生成、搜索与下载不依赖该视频。

## 结论

未发现需要阻塞交付的 P0/P1/P2 问题。完整自动化覆盖见 `npm run test:e2e`：8 项通过。
