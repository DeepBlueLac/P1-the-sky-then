# 此刻星空 Web 重构 PRD

版本：v1.0-draft  
日期：2026-07-14  
状态：已实现并验收
产品域名：<https://the-sky-then-live.vercel.app/>

## 1. 一句话目标

将“此刻星空”重构为一个电影感、全屏、可探索的 Web 星空体验：用户输入时间与地点后，真实天文数据应成为整个页面的视觉主角，而不是被限制在一个小型静态海报卡片中。

## 2. 重构原则

- 只面向 Web，不再维护 iOS、Android、Expo 原生端兼容性。
- 整个前端信息架构、组件结构、视觉语言与交互方式重新设计，不继承初版“左侧长表单 + 右侧小海报”的产品思路。
- 可以保留并复用已经验证的天文计算、地点查询、隐私逻辑、数据文件、PNG 导出规则和自动化测试资产。
- 当前实现只作为功能需求清单，不作为新版视觉或布局参考。
- 核心体验必须先壮观、可读、可探索，再考虑传统表单和设置项的呈现。

## 3. 用户价值

用户不是来“填一张表”的，而是来重新进入某个重要时刻：看见当时当地的真实星空、探索星座、调整纪念文字，并保存一份具有仪式感的数字纪念品。

新版需要传达三种感受：

1. **宇宙尺度感**：第一眼看到的是辽阔、深邃、持续运动的真实星空。
2. **私人纪念感**：时间、地点和题词自然融入体验，不像后台管理表单。
3. **可信与克制**：动态效果服务于真实天文数据，不制作虚假的星座、星位或夸张数据。

## 4. 范围

### 4.1 本期必须完成

- 全屏 Web 首屏与沉浸式星空舞台。
- 日期、当地时间、地点搜索与选择。
- 基于真实地点和时刻生成可见恒星、星座连线与名称。
- 星空的动态进入、指针视差、悬停、点击聚焦、星座显隐和时间微调。
- 纪念标题与题词编辑。
- 独立的作品编辑/导出模式与免费 PNG 下载。
- 桌面、平板和手机浏览器响应式体验。
- 键盘、焦点、减少动态效果和静态降级支持。
- 视觉回归、主流程、API、数据准确性和下载测试。

### 4.2 明确不做

- iOS / Android 原生应用与 Expo 原生兼容。
- 用户账号、云端作品库、真实支付、订单或订阅。
- 社交动态、社区信息流或与产品无关的营销模块。
- 为追求效果而伪造星点、星座线或天文结论。

## 5. 创意方向：The Living Sky

### 5.1 核心画面

页面以一整屏运动星空作为“舞台”，真实计算出的星点覆盖或融合在背景视频之上。玻璃面板只是漂浮在宇宙中的操作层，不能成为视觉主体。

初始状态展示品牌、短标题、时间地点入口和一个主要 CTA。生成后，界面平滑进入“观星模式”：导航与输入缩为边缘控制条，星空扩大到接近全屏，用户可直接探索。

禁止继续使用大块静态黑色矩形承载 `THE NIGHT WE MET`。标题应成为可编辑的电影字幕式排版，悬浮在星空或地平线之上；正式海报排版只在“作品模式”中出现。

### 5.2 视觉完成度目标

参考用户提供的 Bloom liquid-glass 方向，但内容必须转译为“此刻星空”的产品语言：

- 全屏自动播放、循环、静音、`object-cover` 视频背景：  
  `https://d8j0ntlcm91z4.cloudfront.net/user_38xzZboKViGWJOttwIXH07lWA1P/hf_20260315_073750_51473149-4350-4920-ae24-c8214286f323.mp4`
- 视频不可用时退化为具有层次的深黑背景，不影响核心功能。
- 全局严格使用灰阶：背景、文字、玻璃与控件不得出现装饰性色彩。
- 天文数据允许使用极少量接近白色的亮度差异表达星等；不使用蓝、紫、红作为 UI 强调色。
- Poppins 用于界面与显示文字；Source Serif 4 仅用于标题中的斜体、诗性强调或题词。
- 标题字重以 500 为主；大标题使用紧凑字距和宽阔留白。
- 圆角令牌 `--radius: 1rem`，大型舞台和面板可使用 `1.5rem–2.5rem`。
- 图标统一使用 Lucide Web 图标，不使用 emoji、文本符号或手绘 SVG 代替图标。

### 5.3 液态玻璃系统

必须保留两级可复用类：

#### `.liquid-glass`

- `background: rgba(255,255,255,0.01)`
- `background-blend-mode: luminosity`
- `backdrop-filter: blur(4px)`
- 无普通 border；通过 `::before` 渐变描边实现边缘反光。
- `box-shadow: inset 0 1px 1px rgba(255,255,255,0.1)`
- 用于轻量按钮、标签、浮动信息和工具条。

#### `.liquid-glass-strong`

- `backdrop-filter: blur(50px)`
- `box-shadow: 4px 4px 4px rgba(0,0,0,0.05), inset 0 1px 1px rgba(255,255,255,0.15)`
- `::before` 使用更清晰的 0.5/0.2 高光透明度。
- 用于主 CTA、输入抽屉、作品设置和关键确认层。

玻璃效果必须有降级方案；在不支持 `backdrop-filter` 的浏览器中仍需保证文字对比度。

## 6. 页面结构

### 6.1 首屏：进入这一刻

首屏高度至少 `100svh`，不出现传统长表单。

- 顶部导航：品牌标志、“此刻星空 / THE SKY THEN”、声音/动态偏好、关于、菜单。
- 主标题建议：“回到那一刻，仰望同一片星空。”
- Source Serif 4 斜体只强调“那一刻”或“同一片星空”。
- 一行式输入入口：日期、当地时间、地点；点击后展开液态玻璃选择器。
- 主 CTA：“进入这片星空”。
- 可信提示：“真实星位 · 本地计算 · 输入保存在此设备”。
- 背景视频占满视口，真实星点在生成前以低密度待机状态缓慢呼吸。

### 6.2 桌面空间关系

保留 52/48 的设计节奏，但不再等同于“表单/海报”两栏：

- 左侧约 52%：品牌叙事、时间地点入口、主要 CTA、纪念标题。
- 右侧约 48%：开放式星穹焦点区、实时状态和轻量探索提示。
- 真实星图画布可以跨越分栏并延伸至全屏背景，不能被右侧面板边界裁成小卡片。
- 生成完成后允许进入全屏 Focus Mode；界面控件缩至四周，星空成为唯一视觉中心。

### 6.3 观星模式

- 星点从背景深处渐入，根据星等决定尺寸、亮度、景深与闪烁频率。
- 星座连线按路径逐段绘制，不瞬间全部出现。
- 鼠标移动产生非常轻微的相机视差；触控设备使用拖拽探索。
- 悬停恒星时显示微型信息层；点击星座名称后聚焦该星座并降低其他星点亮度。
- 用户可切换：星座线、星座名称、地平线、方向、环境视频。
- 时间滑杆允许在当前时刻前后移动；星位变化必须来自真实重新计算或插值。
- “回到这一刻”按钮恢复用户输入的准确时间。
- 所有动画均支持 `prefers-reduced-motion`；减少动态效果时使用淡入与静态星图。

### 6.4 作品模式

作品模式是独立的全屏编辑层，不常驻首屏：

- 提供沉浸式全幅作品预览，而不是小型黑色海报卡。
- 标题、题词、日期与地点可直接在画面上编辑。
- 提供少量高质量构图模板：极简天文台、编辑部图册、午夜花园。
- 模板只改变排版、纹理和字体关系，不改变真实星位。
- 支持竖版、方形和壁纸画幅；画布在编辑时尽可能占满可用高度。
- 免费 PNG 下载必须直接可见；付费意向不得阻塞免费导出。
- 导出过程显示明确进度、成功和恢复状态。

### 6.5 移动端

- 仍为纯 Web 响应式页面，不开发原生应用。
- 不照搬桌面两栏：首屏先展示标题、紧凑输入和可进入的星空窗口。
- 生成后星空占据主要视口；设置通过底部抽屉进入。
- 触控目标不小于 44×44 CSS px。
- 不出现横向滚动、不可关闭抽屉或被浏览器地址栏遮挡的 CTA。

## 7. API 与数据现状

### 7.1 现有服务器 API

#### `GET /api/health`

当前返回：

```json
{ "ok": true }
```

它只用于部署健康检查，不包含可视化数据。

#### `GET /api/geocode?q=Paris`

该接口代理 Open-Meteo 地理编码服务。Paris 第一条结果当前包含：

```json
{
  "id": 2988507,
  "name": "Paris",
  "latitude": 48.85341,
  "longitude": 2.3488,
  "elevation": 42,
  "feature_code": "PPLC",
  "country": "France",
  "admin1": "Île-de-France Region",
  "timezone": "Europe/Paris",
  "population": 2138551
}
```

新版前端不应原样展示整段 API 数据。应把它转译为地点搜索结果：主标题为城市，次级信息为地区/国家，选择后显示当地时区、坐标和当地时间。人口、邮编等与核心任务无关的字段默认不展示。

### 7.2 真正的星空数据

星空不是服务器 API 返回的。浏览器使用 `astronomy-engine`、恒星目录和星座数据本地执行 `buildSkyMap()`，当前返回：

```ts
type SkyMap = {
  utcDate: Date;
  stars: { x: number; y: number; magnitude: number; opacity: number }[];
  lines: { x1: number; y1: number; x2: number; y2: number }[];
  labels: { x: number; y: number; text: string; rank: number }[];
};
```

这组数据已经足够驱动 SVG、Canvas 2D、PixiJS 或 WebGL 动画。新版应优先在浏览器本地继续计算，以保持即时反馈与隐私，不需要为了动画强行新增 `/api/sky`。

### 7.3 数据增强要求

为了支持更丰富的交互，需扩展投影结果：

```ts
type InteractiveStar = {
  id: string;
  x: number;
  y: number;
  altitude: number;
  azimuth: number;
  magnitude: number;
  opacity: number;
  name?: string;
  constellationId?: string;
  twinkleSeed: number;
};
```

同时需要：

- 为星座连线增加 `constellationId`，支持独立聚焦、显隐和动画。
- 为标签增加中英文全名，不只显示 `UMa`、`Ori` 等缩写。
- 为亮星补充稳定 ID、常用名和所属星座；没有可靠数据时不显示虚构名称。
- 保留 `altitude`、`azimuth`，供地平线、方向和详情层使用。
- 在 Web Worker 中计算不同时刻的星图，避免拖动时间轴时阻塞主线程。

### 7.4 建议的呈现层

- 默认观星模式：WebGL 或高性能 Canvas，承担星点、连线、视差、聚焦和时间过渡。
- 无 WebGL 或减少动态效果：Canvas 2D / SVG 静态降级。
- 作品导出：独立 SVG 或离屏 Canvas，不截取整张网页。
- 地点 API：只负责检索与标准化地点，不参与视觉渲染。
- 可分享链接如后续需要，可把日期、时间、地点 ID、主题和题词编码进 URL；不上传精确位置或私人文字。

## 8. 关键交互状态

- 初始、输入展开、地点搜索中、无结果、地点服务失败。
- 星空计算中、进入完成、观星、星座聚焦、时间移动、恢复原时刻。
- 作品编辑、导出中、下载成功、导出失败可重试。
- 视频加载失败、WebGL 不可用、减少动态效果、离线。
- 所有状态均需要明确文案和恢复路径，不允许只显示旋转加载图标。

## 9. 技术方向

- 前端改为 Web 专用 React + TypeScript 架构；不再以 React Native / Expo 组件作为页面布局基础。
- 保留 Vercel Functions、`astronomy-engine`、`date-fns-tz`、Zod、恒星目录和星座目录。
- 动态星空优先评估 PixiJS 或 Three.js / React Three Fiber；选型以清晰度、可访问降级、包体与导出兼容性为准。
- 星图计算放入 Web Worker；UI 状态与计算状态分离。
- 作品导出使用独立渲染管线，避免背景视频、跨域字体或 DOM 滤镜污染导出。
- 远程视频提供 poster、超时和失败降级，不阻塞首次可交互。
- 所有视觉资源必须有明确来源与许可，不提交密钥、Token 或个人数据。

## 10. 性能与可访问性

- 首屏主要文本与 CTA 不等待视频或星图即可显示。
- 1440×900 桌面上星空视觉区域应覆盖至少 65% 首屏面积；Focus Mode 接近 100%。
- 默认动画目标为 60fps；中低端设备自动降低星点特效与分辨率，不减少真实星点数量。
- 页面缩放 200% 后主流程仍可完成。
- 键盘可完成日期、时间、地点、生成、星座切换和下载。
- 焦点轮廓清晰；玻璃背景上的正文和控件满足 WCAG AA 对比度。
- 动画、视差、自动视频均尊重减少动态效果偏好。
- 视频静音，不自动播放声音。

## 11. 自动化测试要求

- 单元测试：时间转换、可见星过滤、星等映射、星座分组、时间插值。
- API 测试：健康检查、地点查询、超时、无结果和无效查询。
- 组件测试：输入抽屉、地点选择、模式切换、错误恢复。
- Playwright：1440×900、1280×800、768×1024、390×844。
- 验证首屏详情清晰、星空占比、无横向溢出、无控制台错误、键盘焦点与减少动态效果。
- 验证 Paris 选择后坐标、时区和星图同时变化。
- 验证星座聚焦、时间滑杆、恢复原时刻和作品模式。
- 验证 PNG 文件非空、尺寸正确、星点数量符合数据结果且不包含网页背景视频。
