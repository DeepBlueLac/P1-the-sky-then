# 此刻星空 Web 全面重构

## 目标

重写 `https://the-sky-then-live.vercel.app/` 的整个前端。只开发 Web，不保留初版“长表单 + 小海报”思路，不再维护 Expo、iOS 或 Android 兼容。真实星空必须成为全屏、壮观、清晰、可探索的视觉主角。

## 设计依据

1. 完整产品与技术规范：`docs/WEB-REDESIGN-SPEC.md`，开发前必须完整阅读。
2. 重要视觉参考：`D:\Ai-Project\Obsidian\DBL_obsidian\会话上下文\设计prompt.md`，视觉系统约 80% 严格遵守。
3. 当前代码只用于复用天文计算、地点 API、数据、隐私规则和测试，不作为新版布局参考。

## 不可妥协的视觉要求

- 保留指定的全屏静音循环星空视频并提供深色降级背景。
- 严格灰阶 UI；Poppins 用于界面，Source Serif 4 仅用于斜体强调。
- 保留 52/48 视觉节奏，但星图可跨越分栏并进入全屏 Focus Mode。
- 实现 `.liquid-glass` 与 `.liquid-glass-strong` 两级玻璃系统。
- 使用 Lucide Web 图标；所有交互元素有 hover、focus、active 状态。
- 禁止把 `THE NIGHT WE MET` 继续放在呆板黑色矩形卡中；标题应融入星空，作品编辑进入独立全屏模式。
- 桌面首屏星空覆盖至少 65% 面积；生成后可接近 100% 视口。

## 核心体验

- 首屏只保留品牌叙事、紧凑日期/时间/地点入口和“进入这片星空”CTA。
- 生成后进入可交互观星模式：星等亮度、星座路径动画、悬停、星座聚焦、时间滑杆、恢复原时刻。
- 星空由真实计算数据驱动。地点搜索继续使用 `/api/geocode`；星图继续在浏览器本地计算以保护隐私。
- 扩展恒星数据：稳定 ID、名称、星座、海拔角、方位角、星等与动画种子；星座线增加 `constellationId`。
- 计算迁移到 Web Worker；渲染优先评估 PixiJS 或 Three.js / React Three Fiber，并提供 Canvas/SVG 静态降级。
- PNG 使用独立 SVG 或离屏 Canvas 导出，不能截取背景视频或整张网页。
- 移动端使用底部设置抽屉，星空占据主要视口，触控目标至少 44×44px。

## 安全边界

- 不删除用户数据，不提交密码、Token、密钥或个人隐私。
- 不增加账号、支付、订单、社区或 PRD 外功能。
- 不伪造星点、星座线或天文结论。
- 遇到无法安全判断的破坏性操作时停止并说明原因。
